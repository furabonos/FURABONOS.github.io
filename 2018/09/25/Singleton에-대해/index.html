Singleton에 대해

2018-09-25
#swift

Singleton ?

  • 특정 클래스의 인스턴스에 접근할 때 항상 동일한 인스턴스만을 반환하도록 하는 설계 패턴
  • 한번 생성된 이후에는 프로그램이 종료될때까지 메모리에 상주
  • 앱에서 유일하게 하나만 필요한 객체에 사용
  • UIApplication, AppDelegate에 사용

예제코드

  • let으로 선언한 것은 기본적으로 thread-safe 하므로 별도로 Dispatch_once_t가 불필요하다.
    (thread-safe : 멀티 스레드 프로그래밍에서 일반적으로 어떤 함수나 변수, 혹은 객체가 여러 스레드로부터 동시에 접근이 이뤄져도 프로그램의 실행에 문제가 없음을 뜻함. 따라서 동시에 함께 실행되더라도 각 스레드에서의 함수의 수행 결과가 올바르게 나오게 나오는 것
    Dispatch_once_t : App 라이프 사이클에서 단 한번만 실행되도록 보장해 주는 것을 의미하며 Thread Safe을 보장 받기 위해 사용함.)
  • static 전역변수로 선언한 것은 lazy 하게 동작하므로 Singleton을 생성하기 전까지 메모리에 올라가지 않음.
  • 외부에서 인스턴스를 직접 생성하지 못하도록 private로 선언.
  • 맨밑의 코드는 에러가 난다. private로 초기화 선언했기 때문에.

활용예제

Singleton을 활용하는 것중 대표적으로 UserDefaults가 있다.
UserDefaults란 ios에서 데이터를 파일에 저장하기 위해 사용하는 대표적인 클래스중 하나이다. 간단한 정보를 저장하고 불러올때 사용하며 내부적으로 plist파일로 저장한다.
