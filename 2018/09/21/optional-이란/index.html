위의 예시는 값이 없는걸까?
값이 없는게 아니라 빈값이 된다. 값이 없을땐 nil이라 표현한다.
보통의 경우 값이 없을때 nil을 쓰지만, 모든 변수에 nil을 넣을수는 없다. 밑의 예시를 보면 오류가 뜬다.


string에는 nil을 할당할수 없다고 나온다. 위 예시의 오류가 사라지려면 이제 옵셔널이란 개념을 이용해야 한다.

달라진 점을 찾자면 바로 ? 의 사용이다. ? 를 붙힘으로써 name의 값이 있을수도 있고 없을수도 있다고 지정해준것이다.

그런데 한가지, print를 찍어보면 뭔가 우리가 알던 것과 다르게 나온다는것을 알수있다.

기존의 값 앞에 optional이 붙는다. 우리가 이 값을 사용하려면 저 옵셔널을 벗겨내야 사용이 가능하다. 이제 그 방법을 R아보자

Optional Binding

옵셔널 값을 가져오는 대표적인 방법중 하나이다.
먼저 코드를 보자.

  1. if let
    옵셔널 값인 optionalStringValue의 값을 nil인지 아닌지 체크하고, nil이 아니라면 stringValue에 값을 전달하고 그 안에서 사용한다.
  2. guard let
    옵셔널 값인 optionalStringValue 의 값을 체크 후에 nil이 아니라면 stringValue에 값을 전달하고 guard문 뒤의 코드를 실행한다. 값이 nil이라면 guard 문을 통과하지 못한다.
  3. switch
    옵셔널 값인 optionalStringValue 값을 미리 정해놓은 값으로 필터링 할때 사용한다. 기본적으로 case를 사용하여 원하는 값을 정해주고 일치할때 각각의 코드를 구현한다.

Optional Chaining

옵셔널이 현재 nil이 될수도 있는 프로퍼티, 메소드, 서브스크립트를 조회하고 호출하는 과정이다. 옵셔널 체이닝에 값이 있으면 프로퍼티, 메소드, 서브스크립트를 호출하고, nil이면 nil을 반환한다. 여러개를 연결할수 있고, 연결된 어떤 링크가 nil 이면 전체 체인은 실패한다.
